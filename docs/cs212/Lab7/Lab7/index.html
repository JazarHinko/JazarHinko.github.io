Mad Lib

Description

In this lab you will implement a mad lib via a web form and a custom server response. If you have never played a mad lib before, you can play one here.

This lab requires that you first switch to using ExpressJS as your web server software!

Instructions

First, if you haven't already, switch to using ExpressJS.

Next, plan your mad lib. Ensure that it has, at minimum, five slots. Note the types of words that should fill each slot such as plural noun, adjective, etc..

Next create a new webpage accessible at the URL
http://<your_web_site>/cs212/lab7

This page should contain a <form> element with the attributes necessary to cause a POST which is submitted to the same URL that the form is served from; how to do this in the class recording on forms.

This form should have a minimum of five textual input fields. The label of each input element should describe the type of input necessary to fill in a gap in your mad lib. Of course, your form should cointain a submit button as well.

Finally, you should wire up a server.post() route handler in your "server.js" file which should send back a textual response which is the mad lib filled in with the information provided by the user. There is nothing complex to be done in this route handler, just use string interpolation and/or concatenation to insert the information from the form into your mad lib string and pass it off to res.send().
